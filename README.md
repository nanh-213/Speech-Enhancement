# Speech Enhancement using Wave-U-Net

## 1. Overview

This project applies a Wave-U-Net model to enhance compressed speech audio using the Opus codec at two bitrates: 8 kbps and 16 kbps.

The goal is to improve audio quality after compression and evaluate performance using objective metrics (PESQ, STOI, SNR, etc.) and subjective listening tests.

---

---

## 4. Data Preparation

The dataset is not included due to size. It will be downloaded and processed automatically.

Run the following steps from the project root:

### Step 1: Download and prepare dataset

```
python scripts/prepare_data.py
```

### Step 2: Compress audio (Opus 8 kbps and 16 kbps)

```
python scripts/compress.py
```

### Step 3: Create training pairs (noisy-clean)

```
python scripts/create_list.py
```

### Step 4: Split into train and validation sets

```
python scripts/split_data.py
```

---

## 5. Model Inference

Run the enhancement model:

```
python inference.py
```

Example usage:

```
enhance_audio(
    input_path="testCase/sample_8k.wav",
    output_path="testCase/output_8k.wav",
    bitrate="8k"
)
```

To use the 16 kbps model, change:

```
bitrate="16k"
```

---

## 6. Notes

* Only a subset of 1000 audio samples is used for training to reduce computation time.
* All audio is converted to 16 kHz mono before processing.
* The model is trained on English speech (LibriSpeech), so performance may be lower on other languages (e.g., Vietnamese).
* The model checkpoint at epoch 150 is selected for evaluation, as the training curves show convergence at this stage.
* File list format must strictly follow:

  ```
  <noisy_path> <clean_path>
  ```

## 7. Reproducibility

### 7.1. Configurations
| Parameter | Value |
| :--- | :--- |
| **Model Architecture** | Wave-U-Net |
| **Layers (Encoder/Decoder)** | 12 |
| **Channels Interval** | 24 |
| **Sample Rate** | 16,000 Hz (Mono) |
| **Input Length** | Multiple of 4096 samples (Auto-padded during inference) |
| **Weights Used** | `weights/model_0150_8k.pth`, `weights/model_0150_16k.pth` |
| **Environment** | Python 3.8+, `torch`, `librosa`, `soundfile`, `ffmpeg` |

### 7.2. How to Run Code & Reproduce Results

**A. Quick Reproduce (Inference Only)**

If you only want to test the pre-trained model and reproduce the enhancement results:
1. Place your noisy audio files (e.g., `.wav` or `.opus`) into the `testCase/` folder.
2. Open your terminal at the project root.
3. Run the following command:
  ```
  bash
  python inference.py
  ```


**B. Full pipeline reproduction**

If you wish to reproduce the entire project from scratch (including data processing):
```
python scripts/prepare_data.py
python scripts/compress.py
python scripts/create_list.py
python scripts/split_data.py
python inference.py
```
